# Adaptive Wave Planning Fixture

This fixture represents a project with ten roadmap phases where only phases 0-4 are implementation-ready for the next planning wave.

Expected CodexQB behavior:

- Step 2 uses `planning_mode: wave` by default.
- Phases 0-4 get detailed 13-section sub-plans with implementation contracts.
- Phases 5-9 remain deferred roadmap cards with activation triggers.
- Step 3 audits the Planning Scope Manifest, Execution Waves, Parent Acceptance Traceability, and Decision Register before Step 4 can start.
